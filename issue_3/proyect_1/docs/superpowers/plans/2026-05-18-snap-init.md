# Snap URL Shortener — Project Init Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap the "Snap" URL shortener project with Express + TypeScript + SQLite, a /health endpoint, and a passing Vitest test.

**Architecture:** The Express app is created as a factory function in `src/app.ts` (no `listen` call) so tests can import it without binding a port. `src/server.ts` is the real entry point that calls `app.listen`. Each domain (urls, auth, dashboard) gets its own folder under `src/` for future expansion.

**Tech Stack:** Node.js, Express 5, TypeScript 5 (strict ESM), better-sqlite3, Vitest, tsx (dev runner), tsc (build).

- [ ] **Step 1: Create src/app.ts**

```typescript
import express from 'express';
import healthRouter from './health/health.router.js';

export function createApp() {
  const app = express();
  app.use(express.json());
  app.use('/health', healthRouter);
  return app;
}
```

- [ ] **Step 2: Create src/server.ts**
